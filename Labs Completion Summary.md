# ⚠️ Important Reminder

> **This guide reflects real Staff/Principal DevOps responsibilities.**

Each lab should be practised in a real:

* ☁️ AWS Free Tier account
* 🧪 Cloud sandbox environment

Reading the material alone is **not sufficient**. Hands-on implementation is required to build operational confidence and practical expertise.

# 🧭 Recommended Learning Sequence

The order of completion is important because each lab builds upon previous infrastructure and operational concepts.

## Phase 1 — Infrastructure Foundation

Complete:

1. **Lab 01 — Cloud Infrastructure**
2. **Lab 02 — Terraform Infrastructure as Code**

Focus areas:

* Cloud networking
* Infrastructure provisioning
* Infrastructure automation
* State management

---

## Phase 2 — Delivery and Container Platforms

Complete:

3. **Lab 03 — CI/CD**
4. **Lab 04 — Docker / ECS**
5. **Lab 05 — Kubernetes**

Focus areas:

* Automated delivery pipelines
* Container orchestration
* GitOps workflows
* Kubernetes operations

---

## Phase 3 — Platform Engineering and Security

Complete:

6. **Lab 06 — Magento Platform Engineering**
7. **Lab 07 — Observability**
8. **Lab 08 — Security Engineering**

Focus areas:

* Production application operations
* Performance optimisation
* Monitoring and reliability
* Security controls

## Phase 4 — Automation and Developer Enablement
